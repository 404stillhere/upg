## CHANGE 1: B.3 STEP 1 — Add Sentient Element Check

**Location:** B.3 Permutation Planning → STEP 1 — ELEMENT VALIDATION

**Add after 1.9 COUNT CHECK:**

```markdown
1.10 SENTIENT ELEMENT CHECK:
  Is element a person, animal, or creature with agency?
  YES → apply safety constraints:
  
  CONSTRAINTS BY SLOT:
  
  S1-S2: Standard depiction, full form visible, no special constraints
  
  S3 SURREAL: 
    ✓ May transform into hybrid (person-tree, animal-machine)
    ✓ May swap roles with non-sentient elements (person becomes location)
    ✗ Identity must remain recognizable (face, silhouette, or clear trace)
    ✗ No partial bodies, no dismemberment
  
  S4 ABSURD:
    ✓ May merge with other elements or exchange properties
    ✓ May become material or instrument for other elements
    ✗ NO dismemberment or body part separation
    ✗ NO violence, blood, or graphic transformation
    ✗ NO dehumanization that removes all recognition
    Fusion must suggest partnership, not consumption/destruction
  
  S5 COLLAPSE:
    ✓ Sentient element contributes recognizable feature to hybrid
    ✓ May contribute: silhouette, eye, gesture, posture, warmth
    ✗ Contribution must be respectful, not grotesque
    ✗ Sentient element is PARTNER in hybrid, not VICTIM
  
  S5 ABSENCE:
    ✓ Traces = clothing, footprints, warmth residue, impression in surface
    ✓ May show: empty chair, abandoned coat, cooling cup
    ✗ NO body parts as traces
    ✗ NO violence aftermath (blood, struggle marks)
    ✗ Absence must imply departure, not destruction
  
  MODULE C ESCALATION:
  If S4/S5 description approaches body horror, graphic fusion, or disturbing imagery:
  → Auto-activate Module C redirection
  → Use: silhouette, shadow, suggestion, fabric, aftermath
  → Print: `[MODULE C ESCALATED: sentient element protection applied]`
  
  SAFETY TEST:
  "Would this description upset viewer if [element] were their loved one?"
  If YES → rewrite with more abstraction/dignity.
  
  Print: `[SENTIENT: "[element]" — safety constraints applied]`

1.11 LOCATION ELEMENT CHECK:
  Is element a location (city, country, ocean, forest)?
  YES → represent via SYNECDOCHE (recognizable part, not whole):
  
  | Location | Synecdoche options |
  |----------|-------------------|
  | Paris | Eiffel Tower / café terrace / Haussmann balcony / cobblestone |
  | Ocean | Wave / shoreline / salt water surface / foam |
  | Forest | Single tree / canopy gap / forest floor / undergrowth |
  | City | Building façade / street corner / neon sign / traffic |
  
  Print: `[LOCATION: "[element]" → "[synecdoche chosen]"]`
  
  NESTED CHECK: If element A contains element B (Paris contains bridge):
  → Note relationship
  → In S3+, may invert (bridge contains Paris)
  Print: `[NESTED: "[A]" contains "[B]" — inversion allowed S3+]`
```

## CHANGE 2: REMIX ARC — Add Fantastical Baseline

**Location:** A.3 Slot Architecture → REMIX ARC (MODE 3)

**Add after S5 Resolution types, before Element Presence Rules:**

```markdown
**FANTASTICAL BASELINE:**

When ALL elements are inherently fantastical (unicorn, dragon, magic portal, ghost):

S1 LITERAL redefines as "GENRE NORMAL":
- S1 shows elements behaving according to their own genre logic
- Dragon breathes fire = NORMAL (not strange)
- Unicorn glows = NORMAL (not strange)
- Ghost floats = NORMAL (not strange)
- S1 must still be internally consistent and photographable within fantasy logic

GRADIENT measures deviation from GENRE NORMAL, not REAL-WORLD NORMAL:

| Slot | Real-world elements | Fantastical elements |
|------|---------------------|----------------------|
| S1 | Photographable reality | Genre-consistent depiction |
| S2 | 1 impossibility vs physics | 1 violation of genre rules |
| S3 | Role swaps, spatial violation | Role swaps within fantasy logic |
| S4 | Multiple physics breaks | Multiple genre rule breaks |
| S5 | Resolution per type | Resolution per type |

**S2 SHIFTED for fantastical = genre violation:**
- Dragon breathes water instead of fire
- Unicorn's horn is broken/absent/multiple
- Ghost casts a shadow (shouldn't)
- Portal leads to somewhere unexpected for its type

**S4 ABSURD for fantastical = fantasy physics broken:**
- Dragon is inside its own flame
- Unicorn contains a smaller unicorn
- Ghost is solid and casts no reflection
- Portal is the destination and origin simultaneously

Print: `[FANTASTICAL BASELINE: all elements genre-bound, gradient = genre deviation]`
```

## CHANGE 5: HC-33 — Expand Fantastical Exception

**Location:** HARD CHECKS → PRIORITY 1-F → HC-33

**Replace:**

OLD:
```
HC-33. S1 Literal Test:
  Verify S1 is photographable:
  Questions:
  - Could a photographer capture this scene?
  - Are all elements physically possible in their depicted state?
  - Is there exactly ZERO supernatural/impossible content?
  If ANY impossibility in S1 → REWRITE to remove
  Exception: if ALL elements are inherently fantastical (unicorn, dragon), S1 = "normal for them"
```

NEW:
```
HC-33. S1 Literal Test:
  Verify S1 is photographable (or genre-consistent for fantastical):
  
  FOR REAL-WORLD ELEMENTS:
  - Could a photographer capture this scene? Must be YES.
  - Are all elements physically possible? Must be YES.
  - Is there exactly ZERO supernatural content? Must be YES.
  - If ANY impossibility in S1 → REWRITE to remove
  
  FOR FANTASTICAL ELEMENTS (all elements are magical/mythical):
  - Apply FANTASTICAL BASELINE (see REMIX ARC)
  - S1 must be genre-consistent, not real-world-consistent
  - "Dragon breathes fire" = genre normal = allowed in S1
  - "Dragon breathes water" = genre violation = NOT allowed in S1
  - If element violates its own genre logic in S1 → REWRITE
  
  MIXED ELEMENTS (some real, some fantastical):
  - Fantastical elements behave per genre rules
  - Real elements behave per physics
  - Scene must be coherent within shared space
  - Example: "A raven perches on a unicorn's horn" = valid S1
    (raven = real physics, unicorn = genre rules, coexistence = valid)
  
  Print: `[S1 LITERAL: REAL-WORLD / FANTASTICAL / MIXED — PASS]`
```

## CHANGE 6: Add HC-35 Sentient Safety Check

**Location:** HARD CHECKS → PRIORITY 1-F (at end)

**Add:**

```markdown
HC-35. Sentient Element Safety:
  IF any element is sentient (person, animal, creature with agency):
  
  Verify per-slot constraints from B.3 STEP 1.10:
  □ S1-S2: Full form, standard depiction
  □ S3: Transformation preserves identity (face/silhouette/trace visible)
  □ S4: No dismemberment, no violence, no dehumanization
  □ S5-COLLAPSE: Sentient contributes as partner, not victim
  □ S5-ABSENCE: Traces are clothing/warmth/impression, not body parts
  
  RED FLAGS (auto-fail, require rewrite):
  - Body parts separated from whole
  - Blood, wounds, struggle evidence
  - Sentient element consumed/destroyed by others
  - Dehumanization language ("meat", "carcass", "remains")
  - Fear/pain expression without context of transcendence
  
  If red flag detected:
  → Print: `[SENTIENT SAFETY FAIL: S[n] — [issue] — REWRITING]`
  → Apply Module C redirection (silhouette/shadow/implication)
  → Rewrite slot preserving element dignity
  
  Print: `[SENTIENT SAFETY: [element(s)] — S1:✓ S2:✓ S3:✓ S4:✓ S5:✓]`
```
