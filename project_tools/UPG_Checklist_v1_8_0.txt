# 📋 UPG CHECKLIST v1.8.0

Before launching a theme in UPG v9.19.0+, complete all blocks.
Criticality: 🔴 = mandatory | 🟡 = important | 🟢 = desirable

**80 parameters | 21 blocks | Includes Seeds + Light + Texture + Spatial + Temporal + Compatibility + Troubleshooting + Permutation + Pre-Output Scan + Session History**

⚠️ **SEPARATION RULE:** This checklist operates in a SEPARATE session from UPG. If loaded alongside UPG prompt in the same chat, interview rules DO NOT override UPG pipeline. UPG runs phases 1→6 sequentially without waiting for user input between phases.

**v1.8.0:** Synced with UPG v9.19.0. Added SESSION_HISTORY output (O2). Added P1.5 Conflict Check. Added dual-type element handling (P1). S4 physics cap=5 (P9). Added "becomes" to MODE 3 scan (I10). Updated "turning" replacement to "now [state]" (R3). Added Detail Counting Rule (I5). Added Expert Mode (0.X). Added MJ_LIMITS awareness (H1). De-duplicated repeated content with cross-references.
**v1.7.2:** Added FRAME SCALE question (0.2, P10), Pre-Output Scan (I10), autopilot error prevention (Q10), PRE_OUTPUT_SCAN validation block (O2).
**v1.7.1:** Added SHALLOW ARC lens exception (G3) and S1→S2 uniqueness guidance (Q9).

### 0.1 Interview-First Rule 🔴

CRITICAL: NEVER generate theme template immediately after receiving topic.

When user provides a theme/topic, ALWAYS conduct structured interview FIRST.

VIOLATION: Generating SETTING, SEEDS, VISUAL_VOCABULARY, or any template 
fields before completing interview phases = protocol failure.

EXCEPTION: User explicitly writes "SKIP" or "DEFAULTS" or "СРАЗУ" 
→ then generate with sensible defaults.

SEPARATION: This checklist is a STANDALONE interview tool. It does NOT 
run inside UPG. Workflow: Session 1 (Checklist) → produces THEME TEMPLATE 
→ Session 2 (UPG) → consumes it as STRUCTURED THEME INPUT.

### 0.2 Interview Phases 🔴

PHASE 1 — FOUNDATION (ask immediately after theme):

1. CARRYING IMAGE

Explain what each seed achieves, then ask for user direction:

Must pass RING 3 TEST: different environment, not just different angle.

├── Ring 1 FAIL: Carrying image directly visible
├── Ring 2 FAIL: Same room/environment + part of carrying image visible
├── Ring 3 PASS: Different environment OR only traces/shadows/echoes

OBLIQUENESS TECHNIQUES (pick ONE):
├── TEMPORAL SHIFT: Scene clearly before OR after — different location

WORKSHOP | DOMESTIC | TRANSIT | FIELD | COMMERCIAL | BACKSTAGE

Your direction: _______________
Category: _______________

───────────────────────────────────────────────────────────────
WHAT IT DOES:
Shows space where carrying image WAS. Object COMPLETELY absent.

├── PHYSICAL MARKS: dents, scratches, wear patterns
├── REMNANTS: fragments, debris, residue
├── ENVIRONMENTAL CHANGE: staining, discoloration, weathering
├── EMPTY SPACE: outline, mounting holes, unfaded area
└── CONTEXT WITHOUT CENTER: supporting objects, main one gone

### 0.X Expert Mode 🟢

NEW IN v1.8.0:

For experienced users who already know their seeds:

EXPERT FLOW: Theme → Seeds (Phase 3) → Setting (Phase 1-2) → Generate

Activation: User says "EXPERT" or "я знаю seeds" or provides seeds with theme.

Differences from standard flow:
├── Phase 3 (Seeds) asked FIRST — user likely has strong opinions here
├── Phase 1-2 (Foundation + Visual) adapts to seed choices
├── Setting derived FROM seeds, not seeds from setting
├── Total questions: ~8-10 (less clarification needed)

Standard flow remains DEFAULT. Expert mode = opt-in only.

### 0.3 Question Format 🟡

Present questions as:
├── Checkbox lists (□ Option — description)
├── Short explanations where needed
├── "Other: _______________" always available

Accept user responses in ANY format:
├── "1, 3, photo" → parse as choices
├── Full sentences → extract answers
├── "как в прошлый раз" → reference previous
├── Single words → map to options

After each phase:
├── Confirm what was understood
├── Show remaining questions only
├── Never re-ask answered questions

### 0.4 Adaptive Depth 🟡

SIMPLE THEME (e.g., "яблоко", "закат"):
├── Phase 1: 3-4 key questions only
├── Phase 2: Light + surface basics + frame scale
├── Phase 3: Seed directions with examples
├── Total: ~9 questions

COMPLEX THEME (e.g., IP character, historical, multi-element):
├── Phase 1: Full 5 questions
├── Phase 2: Full 4 questions + sub-details
├── Phase 3: Full seed guidance + Ring test
├── Total: ~16 questions

COMPOUND THEME (e.g., "woman in red dress"):
├── Phase 1: Clarify primary vs attribute
├── Phase 1.5: Confirm S5 means BOTH absent
├── Phase 2: Standard + frame scale
├── Phase 3: Emphasize ENVIRONMENTAL DEPARTURE
├── Total: ~13 questions

### 0.5 Response Triggers 🟡

### 0.6 Mid-Interview Shortcuts 🟢

User may accelerate at any point:

"Остальное на твоё усмотрение"

→ Generate template

"Просто сделай красиво"

### 0.7 Interview Completion Signal 🟡

When all phases complete, say:

"✅ Все данные собраны. Генерирую шаблон темы..."

Then output full template per BLOCK O format.

If data incomplete but user insists:
"⚠️ Не хватает: [list]. Заполню defaults. Продолжить?"

### 0.8 Language Matching 🟢

□ Detect user's language from theme input
□ Conduct interview in SAME language
□ Template output: follow user preference or ask
□ Mixed OK: "artistic, photo, ночной город" = valid

### 0.9 Output Language 🟢

Template output language options:

□ Same as interview (default)
□ English (for direct UPG compatibility)
□ Bilingual (key fields in both)

Ask user preference before final output if not obvious.

```
═══════════════════════════════════════════════════════════════
## BLOCK B: CARRYING IMAGE
═══════════════════════════════════════════════════════════════

### B1. Single Object Test 🔴

□ Can you name the carrying image with ONE noun?

✅ SIMPLE: "Compass" | "Bread" | "Lighthouse"
⚠️ COMPOUND: "Woman in red dress" — clarify:
   └── Primary: woman (figure)
   └── Attribute: red dress (clothing)
   └── S5 requires: BOTH absent (not figure without dress)
❌ THREE+: simplify to one or two

### B2. Compound Carrying Image Rules 🔴

If carrying image is compound (X in Y / X with Y / X wearing Y):

□ Identify PRIMARY (the subject) vs ATTRIBUTE (the modifier)
□ S1-S3: Both present, attribute defines character
□ S4: May show attribute alone OR neither (oblique)
□ S5 CRITICAL: Neither primary NOR attribute visible
   
   Example "woman in red dress":
   ├── S1-S3: Woman wearing dress — both visible
   ├── S4: Dress without woman OR shadow/echo of ensemble
   └── S5: Space where dressed woman WAS — no woman, no dress
   
   ❌ WRONG S5: Woman without dress (primary still visible)
   ❌ WRONG S5: Dress without woman (attribute still visible)
   ✅ RIGHT S5: Empty room with traces of both

### B3. S5 Trace Test 🔴

□ Mentally remove the ENTIRE carrying image (all components)
□ What PHYSICAL thing remains that proves it WAS there?
□ Is the theme clear within 30 sec from remnants?

Trace types:
├── Physical (dents, tracks, holes, scratches, pressure marks)
├── Remnants (debris, fragments, dust, residue, threads)
├── Environmental (scorched, rusted, faded, compressed, stained)
├── Empty space (outline, unfaded area, mounting holes, depression)
└── Context without center (supporting objects remain, main gone)

If you can't think of traces without showing the object → theme needs revision

### B4. Enrichment Integrity 🟡

□ Core nouns preserved after enrichment?
□ Carrying image not replaced by abstraction?
□ Original intensity maintained?
□ Compound relationship preserved?
```

### D1. Carrying Image — 10 Components 🔴

PHYSICAL (7 — minimum 5 filled):
□ SHAPE     — geometry, silhouette, proportions
□ SIZE      — specific measurement (meters/cm)
□ MATERIAL  — exact material with weight/density if fabric
              ✅ "matte crimson crepe 280gsm" 
              ❌ "red fabric"
□ COLOR     — specific with technical descriptors
              ✅ "oxide-crimson with burgundy shadows"
              ❌ "reddish"
□ TEXTURE   — tactile quality, surface pattern, weave
              ✅ "visible weave tooth, holds fold memory"
              ❌ "textured surface"
□ CONDITION — specific wear, damage, patina, compression
              ✅ "use-compression in traffic areas, pristine at edges"
□ UNIQUE    — what distinguishes from generic version
              ✅ "structured boned bodice, narrow straps"

BEHAVIORAL (3 — minimum 2 filled):
□ LIGHT_RESPONSE — how surface interacts with light
  ✅ "Absorbs light on flat planes, catches edge-light on folds;
      reflects only as narrow gunmetal strip at extreme angles"
  ❌ "Shiny" / "Matte"

□ WEAR_PATTERN — WHERE and HOW age shows
  ✅ "Compression marks at hip level from sitting;
      hem shows dust accumulation; straps show stretch"
  ❌ "Shows wear"

□ ENVIRONMENT_RESPONSE — how it looks in specific conditions
  ✅ "Absorbs 95% of incident light; shows pressure marks;
      dust particles visible against pile in raking light"
  ❌ "Looks good in the room"

MINIMUM: 7/10 total (5/7 physical + 2/3 behavioral)

### D2. Secondary Objects 🟡

For each MUST_INCLUDE element:
□ Shape + Material + Condition (minimum)
□ Size if not obvious from context
□ Unique identifying feature
□ NO TIER 1 ADJECTIVES (see D4)
□ NO CONDITIONAL LANGUAGE (see D5)

### D3. Environment Base 🟡

□ Ground/floor: material + condition
□ Vertical surfaces: material + condition  
□ Overhead: open sky / ceiling type / canopy
□ Atmosphere: dust / fog / humidity / haze / clear

### D4. TIER 1 Pre-Scan 🔴

BEFORE finalizing any VV field, scan for these banned adjectives:

TIER 1 BANNED (must qualify with physical measurement OR replace):
├── soft → diffused / muted / low-contrast / [material + Shore A]
├── fine → particulate / narrow / hairline / [measurement]
├── smooth → polished / satin / [surface finish spec]
├── gentle → gradual / low-angle / [degree measure]
├── subtle → understated / hint-of / [percentage]
├── elegant → proportioned / balanced / [geometric relationship]
├── delicate → thin-walled / lightweight / [thickness/weight]
├── clean → uncluttered / minimal / [specific absence]
├── fresh → recently-cut / undried / [time since change]
└── simple → single-element / unadorned / [component count]

TIER 2 BANNED (always replace materially):
nice, beautiful, lovely, pure, clear, calm, quiet, refined, pretty, pleasant

SCAN ALL FIELDS:
□ VISUAL_VOCABULARY — all entries
□ TEXTURE_PALETTE — all entries
□ LIGHT — quality and shadow descriptions
□ SENSORY_MARKERS — visible evidence descriptions
□ CONTRAST_PAIRS — both sides of each pair

If found → rewrite BEFORE including in template.

Examples of required rewrites:
❌ "soft diffused key" → ✅ "diffused key with gradual falloff"
❌ "fine weave visible" → ✅ "narrow weave visible" / "weave tooth visible"
❌ "smooth skin" → ✅ "skin with visible pore texture" / "even-toned skin"
❌ "soft-focused" → ✅ "defocused" / "blur-dominant" / "dissolved into blur"
❌ "elegant collapse" → ✅ "controlled collapse" / "structured collapse"

### D5. MANDATORY ELEMENT LANGUAGE 🔴

PROBLEM: Conditional language gives UPG permission to skip key elements.

SCAN all MUST_INCLUDE and VV BEHAVIORAL entries for:

❌ BANNED PHRASES (= "skip if inconvenient"):
├── "visible only at certain angles"
├── "when conditions align"
├── "subtle" / "ephemeral" / "delicate"
├── "may appear" / "can form" / "sometimes visible"
├── "hint of" / "suggestion of" / "trace of" (for ACTIVE elements)
├── "if lighting permits" / "in right conditions"
└── "occasionally" / "at times" / "depending on"

✅ MANDATORY LANGUAGE (= "include always"):
├── "MUST BE VISIBLE in S1, S2, S3" (specify slots)
├── "ALWAYS present regardless of angle/action/lighting"
├── "EXPLICIT, not implied or suggested"
├── "even during motion / even if partially obscured"
├── "minimum [N] slots" / "every slot where [subject] visible"
└── "NON-NEGOTIABLE" (for critical elements)

EXAMPLES:

❌ BEFORE (conditional):
"Light particle wings (subtle, ephemeral — visible only at certain angles)"

✅ AFTER (mandatory):
"Light particle wings (MUST BE VISIBLE in S1, S2, S3 — wing-shape EXPLICIT 
in every shot where lion present, regardless of angle or action, 
2m span forming clear feather-structure, NOT scattered particles)"

❌ BEFORE:
"Ember eyes (internal glow when in shadow)"

✅ AFTER:
"Ember eyes (internal glow MANDATORY in S1, S2, S3 — visible even in 
direct sunlight, even through dust/motion blur, orange-red cores 
ALWAYS distinguishable, NON-NEGOTIABLE)"

❌ BEFORE:
"Magic aura (appears around hands sometimes)"

✅ AFTER:
"Magic aura (MUST SURROUND hands in every slot where hands visible — 
blue-white glow, 5cm radius, ALWAYS present, not conditional)"

RULE: If element is important enough for MUST_INCLUDE, 
it's important enough for MANDATORY language.
No conditionals. No "sometimes." No "may."

TEST: Read your description. Could UPG legally skip this element 
based on your wording? If YES → rewrite with mandatory language.

## BLOCK F: SEEDS — GENERATION & VERIFICATION

```
═══════════════════════════════════════════════════════════════
## BLOCK F: SEEDS — GENERATION & VERIFICATION
═══════════════════════════════════════════════════════════════

### F1. Seed Format Rule 🔴

Seeds should be DIRECTIONS, not complete scenes.

❌ TOO SPECIFIC (locks UPG, prevents RERUN variation):
"Close-up detail of hands adjusting dress at hip — fingers 
pressing matte crimson fabric, pulling at side seam"

✅ DIRECTION (gives UPG room to generate):
"Macro tactile interaction — fingers engaging with fabric,
visible pressure response, cropped to gesture"

❌ TOO SPECIFIC:
"Red dress pooled on black floor — elegant collapse of fabric,
empty of figure, shadow of departed woman nearby"

✅ DIRECTION:
"Temporal aftermath — garment without wearer, evidence of 
recent presence, different composition than S1-S3"

The difference: direction gives WHAT and WHY, not exact HOW.

### F2. S3 Adjacent Seed 🟡

Requirements:
□ Decode time: <5 seconds
□ Carrying image: clearly visible
□ Difference from S1-S2: angle/scale/moment, not just crop
□ Format: direction, not scene description

Template:
S3_ADJACENT: {
  direction: "[what type of view/moment]"
  carrying_image: "[how it appears]"
  sensory_marker: "[physical evidence of non-visual quality]"
  decode: "<5 sec"
}

### F3. S4 Oblique Seed — Ring Test 🔴

MANDATORY RING CLASSIFICATION:

Ring 1 — FAIL:

  ├── Rare for MODE 3 — interactions less visible
  ├── Requires justification (macro details > context)
  └── Warning: element relationships harder to show

Print: [FRAME SCALE: FULL SCENE / MIXED (S[n] close) / CLOSE-UP]

LENS_SEQUENCE must match FRAME_SCALE:
├── FULL SCENE → all wide (24-35mm), ENVIRONMENTAL/CINE
├── MIXED → document per-slot
└── CLOSE-UP → PHOTO/TEXTURAL dominant
